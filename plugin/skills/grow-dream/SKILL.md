## 回顾维度

| 维度 | 检查内容 |
|------|---------|
| **skill** | 对话中暴露了哪些 skill 的缺失、错误或不足？触发关键词是否需要更新？ |
| **command** | 是否有重复 ≥2 次的确定性操作序列？能否用一条命令替代手工步骤？ |
| **rule** | 对话中暴露出哪些新的规则需要？现有规则是否有错误或需要增强？ |
| **agent** | 是否有适合做成专用 agent 的重复操作模式？格式为 `.claude/agents/<name>.md`（YAML frontmatter + Markdown 体） |
| **hook** | 是否有适合做成 hook 的自动化行为？评估触发条件、时机、效率、失败影响 |
| **CLAUDE.md** | 是否存在？内容与项目实际架构是否一致？是否有过期路径/命令/约定？ |
| **AGENTS.md / agents/** | 是否存在？定义的 agent 角色是否匹配项目需求？ |
| **memory** | 频次 ≥2 的规律提醒或跨会话用户偏好？已有条目是否覆盖？MEMORY.md 同步？ |
| **[交叉检查]** | 各文件间术语/描述是否一致？用户纠正过的行为是否已更新到对应文件？ |

## 执行步骤

0. **确定输入源** — 选择分析范围：本次对话（默认）/ 对话日志 / git log / memory/

1. 浏览对话，识别重复发生的模式、错误、或显式纠正

2. 扫描项目 CLAUDE.md、AGENTS.md、`.claude/agents/`、`.claude/rules/` 及 memory/，与发现的实际行为对照

3. **交叉检查**：逐对各文件对同一概念的描述是否一致；用户纠正过的行为是否已更新

4. **提炼分类** — 按频次和性质归类：
   - 重复 ≥2 次的确定性操作 → command
   - 特定场景需判断分支 → skill
   - 跨项目通用行为约束 → rule
   - 独立判断、领域限定 → agent
   - 事件触发自动化 → hook
   - **频次 ≥2 规律提醒、跨会话偏好** → **memory**

5. 对每个维度逐一判断是否有改进空间

6. 输出结构化建议

7. **可选执行**：询问用户是否直接产出改进文件

8. **grill-me 追问验收** — 调用 grill-me 对改进建议进行交叉拷打：
   - 针对每个 **高频模式候选**：使用频次？跨会话适用？值得固化还是 shelfware？
   - 针对每个 **memory 候选**：是否满足"频次≥2 + 跨会话"门槛？已有条目是否已覆盖？
   - 针对每个 **skill/command 候选**：触发条件够明确？维护成本可接受？会不会误触发？
   - 对每个候选输出：**建议采纳** / **建议放弃** / **需补充信息**
   - grill-me 的判定追加到输出末尾，作为最终验收结论

## 输出格式

```
## 改进建议

### skill / command / rule / agent / hook / CLAUDE.md / AGENTS.md / memory / 交叉检查 / 项目文档
- [name]: 建议 + 原因

## grill-me 验收

对每个候选逐一追问判定：

### 高频模式
- [模式描述]: 采纳/放弃/需补充 — 理由

### memory 候选
- [条目]: 采纳/放弃/需补充 — 频次、跨会话适用性

### skill/command 候选
- [name]: 采纳/放弃/需补充 — 触发条件、维护成本
```

## 执行模式（可选）

用户确认后，直接产出改进文件：

| 产出类型 | 目标路径 | 适用条件 |
|---------|---------|---------|
| **skill** | `plugin/skills/<name>/SKILL.md` | 有判断分支、需要上下文理解 |
| **command** | `plugin/skills/<name>/SKILL.md` | 步骤固定、无需 AI 判断 |
| **rule** | `.claude/rules/<name>.md` | 跨项目的通用行为约束 |
| **agent** | `.claude/agents/<name>.md` | 持续运行、有决策自主权 |
| **hook** | `.claude/hook.<event>.sh` | 事件触发、无交互的自动化 |
| **memory** | `memory/<name>.md` + 更新 `MEMORY.md` | 频次 ≥2、跨会话可复用的规律/偏好 |

## 入口图要求

所有产出文件的入口必须包含 **三个 ASCII 图**：管道图 (Pipeline)、链路图 (Link Diagram)、矩阵图 (Matrix Flow)。违反视为输出不完整。

## 输出检查

- [ ] 入口图：管道图 + 链路图 + 矩阵图
- [ ] 产出物可直接使用（路径、格式正确）
- [ ] 去除了对话中的试错痕迹
- [ ] 触发条件明确
- [ ] 不包含时间敏感信息
- [ ] 术语与项目统一
- [ ] **memory 完整性**：频次 ≥2 的规律已写入？MEMORY.md 索引同步？
- [ ] **grill-me 验收**：每个改进候选已过 grill-me 追问，给出明确判定
