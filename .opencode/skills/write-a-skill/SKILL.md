# 写 Skill

## 流程

1. **收集需求** — 问用户：
   - 这个 skill 覆盖什么任务/领域？
   - 处理哪些具体场景？
   - 需要脚本还是只有指令？
   - 需要引用什么资料？

2. **起草 skill** — 创建：
   - SKILL.md 精简指令
   - 内容超 500 行则拆参考文件
   - 确定性的操作写脚本

3. **评审** — 给用户看：
   - 覆盖使用场景了吗？
   - 缺什么？
   - 需要加例子吗？

## Skill 结构

```
skill-name/
├── SKILL.md           # 主指令（必须）
├── REFERENCE.md       # 详细文档（按需）
├── EXAMPLES.md        # 使用示例（按需）
└── scripts/           # 工具脚本（按需）
    └── helper.js
```

## Description 要求

description 是 AI 决定加载哪个 skill 的唯一依据。

**目标**：给 AI 足够信息判断：
1. 这个 skill 提供什么能力
2. 什么时候触发（关键词、上下文、文件类型）

**格式**：
- 最多 1024 字
- 第一句：做什么
- 第二句："用户说 XXX 时触发"

## 什么时候加脚本

- 操作确定性的（验证、格式化）
- 相同代码反复生成
- 错误需要显式处理

脚本比生成的代码省 token、更可靠。

## 什么时候拆文件

- SKILL.md 超 100 行
- 内容有清晰分领域
- 高级功能很少用

## 核查清单

- [ ] description 包含触发条件
- [ ] SKILL.md 100 行以内
- [ ] 没有时间敏感信息
- [ ] 术语一致
- [ ] 有具体例子
- [ ] 引用不超过一层
