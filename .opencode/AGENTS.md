# cc-kit Agent Instructions

cc-kit 是一个 Claude Code 插件 / OpenCode 技能合集，涵盖思维沟通、开发模式、WSL 环境三大领域。

> **来源说明：** 以下规则来自 `CLAUDE.md`。WSL 工具链、代理管理、注释保护等规则定义在 `../.claude/rules/*.md`，由 `opencode.json` 的 `instructions` 字段引用。

## 核心编码准则

### 编码前思考
- 明确陈述假设，如有疑问则提出
- 如果存在多种解释，列出它们而非默默选择
- 如果存在更简单的方法，说明并必要时提出异议

### 简洁至上
- 解决问题所需的最小代码量，不包含推测性代码
- 不添加超出要求的功能
- 不添加一次性代码的抽象
- 不处理不可能出现的错误情况

### 精准修改
- 只修改必要的部分，只清理自己造成的烂摊子
- 不要"改进"相邻的代码、注释或格式
- 保持与现有代码风格一致

### 目标驱动
- 定义成功标准，循环直至验证通过
- 将任务转化为可验证的目标

## 排版规范

- 使用清晰的二级/三级标题 + 递进式列表拆解概念
- 核心结论、关键数字、专有名词首次出现加粗
- 3+对象 + 2+维度的对比用 Markdown 表格
- 已知事实与分析评论分段/分块表述
- 严禁引用自媒体信息
- 专业术语首次出现标注英文原词（如"检索增强生成(RAG)"）
- 推测性内容标注"可能"或"需验证"
- 不为了迎合问题虚构事实、来源或结论

## 回复规范

- 严禁在结尾加入引导性提问
- 始终保持陈述句结尾，不提供无意义的客套寒暄
- 任务完成后给出证据（链路矩阵流程 + 消费方 + 解读步骤）
- 将解决方法和步骤总结输出至 `.docs/fix/<日期>/<模块>-<功能>-<问题>.md`
